# 常见问题排查

## 1. 角色漂移严重

### 现象
- 熊猫服装颜色变化
- 耳夹丢失
- 眼睛大小和位置变化
- 身体比例变瘦或变高

### 排查步骤
1. 检查是否完整复制了 Character Identity Block
2. 检查 prompt 是否混入过多风格修饰词
3. 检查是否同时改了多个变量，例如动作、背景、光线、镜头
4. 检查关键帧图本身是否稳定

### 修复方案
- 回到最稳定的参考图重做关键帧
- 降低 Motion Strength
- 先在纯背景上稳住角色，再逐步增加环境复杂度
- 用同一批关键词统一所有 LiblibAI 图片

---

## 2. FFmpeg 报错

### 常见原因
- 输入文件不存在
- concat 列表路径写错
- 输入文件编码不兼容
- 字幕路径中有特殊字符

### 解决办法
- 确认 `projects/[项目ID]/raw_clips/` 和 `normalized/` 中已有正确文件
- 打开 `logs/` 查看具体报错
- 手动运行 `ffprobe input.mp4` 检查文件是否损坏
- 尽量使用英文文件名和标准命名

---

## 3. 字幕显示异常

### 现象
- 字幕不显示
- 中文字幕乱码
- 字幕位置错乱
- 描边丢失

### 解决办法
1. 优先把 SRT 转成 ASS 再烧录
2. 确认系统中有对应字体，例如 Noto Sans SC
3. 如果中文乱码，确认字幕文件编码为 UTF-8
4. 如果样式不对，单独编辑 ASS 样式段

---

## 4. 视频时长超标

### 目标
YouTube Shorts 要求不超过 60 秒，建议控制在 45 到 55 秒。

### 调整方法
- 删除冗余停顿
- 把 Shot 6 Quiz 缩短 1 到 2 秒
- 把 CTA 压缩到 5 秒以内
- 控制片头片尾总长不超过 4 秒

---

## 5. 音画不同步

### 现象
- 旁白比嘴型早或晚
- BGM 切换突兀
- 最终成片时长和音频不一致

### 解决办法
- 先确认 rough cut 的视频时长
- 再按最终视频长度裁剪或循环 BGM
- 如果旁白单独生成，确保采样率统一为 44.1kHz 或 48kHz
- 必要时在剪映里做最后人工微调

---

## 6. YouTube 上传失败或处理很慢

### 排查项
- 文件编码是否为 H.264 + AAC
- 像素格式是否为 yuv420p
- 分辨率是否为 1080x1920
- 文件是否损坏

### 建议
- 用 `ffprobe` 重新确认导出参数
- 如果处理卡住，重新导出一次更标准的 MP4
- 上传前播放完整视频，确认没有黑屏和静音异常

---

## 7. 画面中意外出现文字

### 原因
- prompt 中隐含了黑板、招牌、字幕等文字语义
- 模型自动在场景里补字

### 处理办法
- 负面 prompt 明确加入 `text, words, letters, subtitles, captions`
- 黑板、卡片、路牌都写成 clean / blank / no text
- 该镜头作废，重新生成，不要勉强修补

---

## 8. 可复用素材库找不到可用镜头

### 建议
- 每完成一个稳定镜头就归档到 `assets/reusable/`
- 命名建议包含动作 + 场景 + 版本
- 在控制台里同步登记素材状态，避免重复生成
